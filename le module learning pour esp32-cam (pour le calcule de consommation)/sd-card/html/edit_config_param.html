Post-traitement
PreValueUse Activer pour utiliser la valeur de lecture précédente pour les vérifications de cohérence - également lors des redémarrages
PreValueAgeStartup Temps (en minutes), combien de temps une valeur lue précédente est valide après le redémarrage (par défaut = 720 min)
AllowNegativeRates Définir sur "false" pour s'assurer que seules les modifications positives sont acceptées (généralement pour le compteur)
Message d'erreur Ne pas afficher le message d'erreur dans la valeur de retour - en cas d'erreur, le dernier numéro valide sera envoyé
Vérifier la cohérence d'augmentation des chiffres Activer la vérification de cohérence supplémentaire - en particulier la vérification du passage à zéro entre les chiffres

Paramètres individuels de post-traitement :
DecimalShift Décaler le séparateur de chiffres dans les chiffres numériques (positif et négatif)
MaxRateValue Changement maximum d'une lecture - si menacé comme changement absolu ou relatif, voir le paramètre suivant.
MaxRateType Définit si le taux de changement par rapport à la valeur précédente est calculé en tant que changement absolu (AbsoluteChange) ou comme taux normalisé à l'intervalle (RateChange = changement/minute).
ExtendedResolution Activer pour utiliser la résolution du point après pour le dernier compteur analogique
IgnoreLeadingNaN Les "N" en tête seront supprimés avant un traitement ultérieur

où les paramètres sont : valeur, taux, horodatage, erreur
